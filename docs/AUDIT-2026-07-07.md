# Full Codebase Audit — 2026-07-07 (v0.50.1)

Multi-agent audit of every source file (~24k lines) against the project's stated direction:
**tool-first memory** — the main model pulls facts on demand via `search_memory` /
`remember_fact` instead of paying for an always-injected fact block every turn.

**Method:** 8 parallel subsystem reviewers (orchestration, extraction agents, generation
agents, storage, retrieval, UX, docs, architecture), every finding required to cite
`file:line` evidence actually read; critical/high findings adversarially verified by
independent skeptic agents (0 refuted; a few severities adjusted one level down).
Static review only — the repo itself notes it is not runtime-tested inside SillyTavern.

**Headline verdict:** the engineering quality is far above typical ST-extension level —
several subsystems genuinely exceed mem0/Letta/Graphiti/Zep (see § State of the art).
The problems cluster in three places:

1. **The tool-first pivot is half-landed.** Push-mode machinery still runs (and bills)
   underneath the tool path in every mode; `tool-only` mode is cosmetic.
2. **Silent-failure edges** in extraction and persistence can lose facts without any error.
3. **Retired features still ship** (~1,500+ lines of dead/dormant code + hidden-but-wired UI),
   and README documents the pre-pivot architecture.

A note on the token model this project is built around: **tool calls are not free.**
Tool schemas ride every request, and each tool call re-bills the entire prompt through the
host's tool loop. What tool-first buys is *conditional* spend — pay only when the model asks —
not zero spend. Several findings below (F-ARCH-3, F-ARCH-5) exist because the code and
prompts assume otherwise.

## State of the art (keep these — they beat the reference systems)

- **Swipe-aware memory** — settle-debounced single extraction per heavily-swiped turn, draft-less
  re-injection cache, tri-state `bf_mem_processed` watermark with `reachedCommit` integrity
  boundary. mem0/Letta/Graphiti/Zep have no concept of regeneration/branching at all.
- **Capture-at-write pinning** — DB profile + character avatar pinned at extraction start,
  re-checked after the LLM await; mid-extraction character switches can't contaminate stores.
- **`search_memory` recall cascade** — exact-handle precision pull → keyword index → trigram
  fuzzy → bounded depth-2 graph walk, tier-first ranking, spoiler-safe no-match hints, scene/
  relationship intents. Ahead of mem0's search API and Letta's archival search.
- **LLM transport hardening** (`llm-call.js`) — per-call AbortControllers in a global registry,
  per-leg fetch-aborting timeouts, 45s wall-clock budget, non-retryable classification,
  cache-drift guard, profile safety designed out rather than patched.
- **Hybrid persistence discipline** — logical version stamps (not upload time), shrink-refusing
  clobber guard, `reconcileDeletes:false` at flush boundaries, before/after fact-census logging.
- **Deterministic supersession + bi-temporal windows** — faithful Graphiti-style
  `valid_from`/`valid_until` port with genesis first-wins provenance, no LLM in the conflict loop.
- **Deterministic retrieval** — `Math.random` genuinely gone; MMR + fixed caps + threshold
  track-reach; swipe-reproducible (modulo F-RETR-1).
- **Anti-hub expansion budget** — per-seed fan-out caps, "connectedness gates candidacy, never
  ranking" invariant. Better-articulated than Graphiti/Zep centrality-boosted traversal.
- **Retrieval observability** — per-candidate exclusion ledger (RECENCY_CUTOFF, CAP_TOKENS…)
  + `explainFactRetrieval` why-not probe. None of the reference systems ship this.
- **Cache-stable prompt discipline** — byte-stable system prefixes with a prefix-hash regression
  canary; scoped dedup candidates (cap 60) so a 10k-fact store costs like an empty one.
- **Token/injection/tool observability UI** — per-agent tokens vs counterfactual baseline,
  injection viewer, per-turn tool audit with runaway softcap. Best-in-class for end users.
- **Finder select-by-identity parsing** — the LLM can only *select* stored facts, never mint
  one; values re-read from the store, never from the LLM echo.
- **ELI5 settings hints** — plain-words + worked example + "just leave the defaults" pattern.
- **CHANGELOG/UPGRADES accuracy** — every spot-checked claim matched code (README is the outlier).

## Recommended roadmap (not in this PR)

1. **Finish the pivot (highest leverage):** make `tool-only` real — gate
   `injectMemoryContext` on mode; cap/budget Unsorted injection and route `remember_fact`
   pins to a ranked category (F-ARCH-1/2). Rewrite the tool descriptions for honest economics
   and add a tool-round-trip term to `netIn` (F-ARCH-3/4).
2. **Close the silent-loss edges:** missing-`#MEM` → `result.error` (F-SCRIBE-1);
   reschedule busy extractions + scan-back for unprocessed exchanges (F-ORCH-3);
   ref-count `isInternalCall` (F-ORCH-2); transactional IDB update helper (F-STOR-2);
   snapshot retry + flush-awaits-in-flight (F-STOR-3).
3. **Delete the retired code** (~1,500+ lines): Finder block + module + UI, vector stack,
   dead embedding chain, hidden embeddings UI, legacy full-scan twins, inert keys.
4. **Unicode tokenization** (F-RETR-2) — one regex change in three places plus a caseless-script
   fallback; biggest reach win available.
5. **Honest review UX:** real pre-commit gate or real Delete; popup off-switch (F-UX-1/6).
6. **README rewrite** around the hybrid default (F-DOCS-1) — partially done in this PR.
7. **True graph memory (if desired):** typed (subject, predicate, object, valid_from,
   invalid_at) edges emitted by the Scribe; `search_memory` traversal by relation type
   (F-ARCH-7). The bi-temporal fact fields are the right substrate to build on.
   — *Shipped 2026-07-09 as the opt-in `typedEdges` feature (default OFF, UPGRADES.md §9): Scribe-emitted `rel:<predicate>@<Category/key>` edges, additive upsert merge, anti-hub graph expansion, and relation-intent matching in `search_memory`.*

## Coverage caveats

Reviewers each declared what they did not reach: no runtime testing inside SillyTavern
(static analysis only), CHANGELOG entries older than v0.23, the spiderweb force-layout
internals (skimmed), `dbProfiles` snapshot layer, and cross-file concurrency between
`remember_fact` and the Scribe on the same category DB (plausible race, unverified —
see F-STOR-2 for the underlying transaction gap). A dedicated cross-file dead-code sweep
agent failed on a session limit; dead-code findings above come from the five subsystem
reviewers and were grep-verified individually.
