🧠 What is the Writer? — click for a simple explanation + example

In plain words: the “Writer” is just your main AI writing the reply. These settings only change what it sees while it writes — how much of the chat history, plus the memory facts the extension hands it.

Example: you’re 200 messages into a story. Instead of making the AI re-read all 200, you can show it just the last few messages plus the key facts (“Kael is a blacksmith, allergic to peanuts”). Same memory, fewer tokens, sharper focus.

Just want it to work? Leave everything at its default — only touch these to save tokens or fine-tune.

Writer (main model)
The Writer is your MAIN model writing the reply — it always uses your default/active connection profile. There is no separate profile to set.
Context
Default 0 = off (main model sees full chat as ST sends it). Set > 0 to HIDE older messages — the main model only sees the last N chat messages + your injected facts. Cleaner focus, lower token cost, but the facts must be good enough to replace the hidden history.
Default 0 = off. When > 0, a new turn REUSES the previous turn's retrieved facts instead of re-running the Drafter + Librarian — for up to N turns before a fresh pull. Cuts tokens/latency and keeps the injected block stable for prompt caching. Memory still updates every turn (the Scribe runs after each reply); only the injected facts may be up to N turns old.
Recall Tool (pull-detail)
Off by default. When on, registers an optional search_memory tool the Writer can call mid-reply to pull a stored fact that wasn't already injected (read-only — never writes). Requires a tool-calling-capable main model; only active on the main generation path. Off by default. When on, registers an optional remember_fact tool the Writer can call mid-reply to PIN one durable fact directly into memory (add-only — never deletes; routes through the same write path as extraction). Requires a tool-calling-capable main model; only active on the main generation path.
Big Picture (summary pyramid)
Off by default. When on, prepends a compact, token-capped overview to the Writer's context: the rolling story summary plus the SHORT per-shelf (category/aspect) summaries relevant to the current scene — a cheap zoom-out instead of hundreds of raw facts. The Writer still drills into specifics via the search_memory tool. Shelf summaries are built periodically during the reflection pass (only for changed buckets), so leaving this on adds no extra generation cost — only a small bounded prompt block.
Moment echo (relationship resonance)
Off by default. When on, on the rare turn a couple's present moment echoes an earlier beat, prepends ONE tiny [Echo: …] line — a single past moment for the two characters present, cued by a reflection-authored callback link or the most-recent charged beat for that pair (never by shared place). Capped at one echo, token-clamped, and emits nothing on most turns. The full relationship history stays pull-only via the search_memory tool — this is just the one-line nudge.
Automatic associative linking
On by default. When a fact is written, it is automatically linked to related existing facts — same subject, shared location/participants, or strong word overlap — so asking about any one surfaces the others. Fully deterministic and free (no LLM call). Links are added to the fact's relationship list (capped, never overwriting existing links).
Retrieval & extraction enhancements
On by default. When choosing which overflow facts to inject, avoids stacking several near-duplicates — it balances importance against being different from what's already chosen, so the same token budget covers more ground. Deterministic (stable across swipes/regens). Turn off for pure importance/recency order.
On by default. Facts the Scribe marked low/medium confidence lose ground to solid facts when competing for scarce injection slots. Never affects a direct/exact match — only the ranking of overflow candidates. Turn off to ignore confidence in ranking.
On by default. Tells the Scribe to convert "yesterday"/"last week" into absolute dates (relative to when the message was sent) so time references still make sense much later. The message's send-time is shown to the Scribe in its data block, not baked into the cacheable system prompt.
Trim Already-Extracted Context
Off by default. Saves input tokens: prior context messages the Scribe already extracted on an earlier turn aren't sent again (their facts are already stored). The new message being analysed and any not-yet-extracted context are always sent — no memory is lost. Turn on to cut Scribe cost on long chats.
Bi-temporal fact validity
Off by default. When on, the Scribe may tag a fact with story-world validity — | from:<when> and | until:<when> (a free-form in-story date, age, or era) — recorded separately from when the fact was learned. When a state fact changes, the old value's story-world window is closed automatically. Keeps flashbacks and time-skips consistent. Purely additive: facts written without it are unaffected.
User-level shared memory
Off by default. When on, facts whose subject is you (your persona / {{user}}) are also saved into one shared global store, and merged into every character's memory on read — so each character remembers the player without re-learning. Your character's own version of a fact always wins on conflict. Purely additive: when off, storage and retrieval are unchanged.
Wipes the shared "facts about you" pool (IDB working copy + backup files). Each character's own memory is untouched. Cannot be undone.
Sequence Depth Dice
For tracked sequences (e.g. a character's location over time) the current step is always included. These set the chance of reaching further back in history. The furthest successful roll sets how far back to include — and every step in between is always included (no gaps).

📝 What is the Scribe? — click for a simple explanation + example

In plain words: the “Scribe” is the note-taker. After each reply, it reads what just happened and saves the important facts into memory — automatically, in the background. It runs after the reply, so it never slows down your message.

Example: Mira says “I promise I’ll guard the forge tonight.” The Scribe quietly writes a note — Events/forge-watch = “Mira promised to guard the forge tonight” — so 50 messages later the AI still knows about the promise.

Just want it to work? Leave the defaults. These settings only change how it takes notes (e.g. which model does it, or asking you to review before saving).

Character Registry

Re-evaluation / Reflection
Every N turns, runs a maintenance pass that DEDUPES the fact database (merging near-duplicate facts) and makes ONE extra LLM call (reusing the Scribe's profile) to write higher-order observation facts. Conservative + cost-aware — it runs infrequently, off the critical path, and never blocks the reply.
Successful pipeline runs between reflection passes. Higher = cheaper, less frequent maintenance.
