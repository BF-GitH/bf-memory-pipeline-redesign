🧠 What is the Writer? — click for a simple explanation + example

In plain words: the “Writer” is just your main AI writing the reply. These settings only change what it sees while it writes — how much of the chat history, plus the memory facts the extension hands it.

Example: you’re 200 messages into a story. Instead of making the AI re-read all 200, you can show it just the last few messages plus the key facts (“Kael is a blacksmith, allergic to peanuts”). Same memory, fewer tokens, sharper focus.

Just want it to work? Leave everything at its default — only touch these to save tokens or fine-tune.

Writer (main model)
The "Writer" is just your normal AI that writes the story reply. It always uses your usual AI, so there's nothing to pick here.
Context
How many recent messages the AI sees. 0 = off (it sees the whole chat, the normal way). Set a number to show only the last few messages plus your saved facts — cheaper and more focused, but only worth it if your facts are good. E.g. set 10 to show just the last 10 messages.
Reuses the same set of saved facts for several replies in a row instead of looking them up fresh every time. 0 = off (look up fresh every reply). Saves time and money, and is gentler on the AI. E.g. set 3 to reuse the same facts for 3 replies. Your memory still keeps growing every turn — only the facts shown to the AI are held steady for a bit.
Recall Tool (pull-detail)
Off by default. Lets the AI look something up in your memory itself, mid-reply, if it needs a fact it wasn't already handed — like checking a notebook. It only reads, never changes anything. Only works with AIs that can use tools (e.g. Claude). E.g. mid-scene the AI looks up "what's Mira afraid of?" on its own. Off by default. Lets the AI jot down an important fact into memory itself, mid-reply, when something worth remembering happens. It only adds notes, never deletes them. Only works with AIs that can use tools (e.g. Claude). E.g. the AI saves "you gave Mira your ring" on its own.
Big Picture (summary pyramid)
Off by default. Gives the AI a short "story so far" recap plus quick summaries of the topics in play, instead of dumping hundreds of separate facts — a bird's-eye view. It stays small so it won't cost much, and the AI can still dig into details when it needs to. E.g. "Chapter so far: you and Mira escaped the city and are now hiding in the woods."
Moment echo (relationship resonance)
Off by default. Once in a while, when the current moment between two characters mirrors something emotional from their past, it slips in one tiny reminder of that earlier moment — for a "remember when..." feeling. It's just one short line, and only on rare fitting turns. E.g. [Echo: the last time you held her hand was at the harbor.]
Automatic associative linking
On by default. When a new fact is saved, it's automatically tied to related facts — same person, same place, or similar wording — so pulling up one brings its friends along. It's free (no AI call) and never erases existing links. E.g. saving "Mira fears fire" connects it to "Mira's house burned down."
Retrieval & extraction enhancements
On by default. Stops the AI from being shown five facts that all say almost the same thing. It mixes in variety so the limited space covers more ground. E.g. instead of three "Mira likes tea" notes, you also get her job and her fear. Turn off to just go by importance.
On by default. Facts the note-taker wasn't sure about get shown less often than solid, confirmed facts when space is tight. An exact match is always shown anyway. E.g. "Mira maybe has a brother" loses to "Mira is a blacksmith." Turn off to treat all facts equally.
On by default. Turns vague time words into real dates when saving a fact, so they still make sense later. E.g. "Mira left yesterday" gets saved as the actual date, so 100 messages on the AI doesn't think it was literally yesterday.
Trim Already-Extracted Context
Off by default. Saves money: the note-taker won't re-read old messages it already pulled facts from. The newest messages are always read, so nothing gets missed. Turn on to lower costs on long chats.
Bi-temporal fact validity
Off by default. Lets memory track when a fact is true inside the story, not just when you learned it. Helps a lot with flashbacks and time-skips. E.g. it can note "Mira was a soldier until age 20, then became a baker," so the AI doesn't mix up past and present.
User-level shared memory
Off by default. Facts about you (your persona) get shared with every character you chat with, so a brand-new character already knows the basics about you. E.g. once any character learns "you're afraid of heights," they all know it. If a character has its own version of a fact, that one wins.
Permanently erases the shared "facts about you" pool. Each character's own memory stays safe. This can't be undone.
Sequence Depth Dice
For things that change over time (like where a character has been), the AI always sees the latest step. These dials set the chance of also showing older steps, so sometimes it remembers more history, sometimes less. E.g. a 70% chance to also recall where Mira was one step ago. The furthest step it reaches back to fills in everything in between — no gaps.
Writer Injection Format
The little wrapper text placed around the saved facts before they're handed to the AI. Advanced — most people never touch this. {facts} gets replaced with the facts and {draft} with the planner's notes.

📝 What is the Scribe? — click for a simple explanation + example

In plain words: the “Scribe” is the note-taker. After each reply, it reads what just happened and saves the important facts into memory — automatically, in the background. It runs after the reply, so it never slows down your message.

Example: Mira says “I promise I’ll guard the forge tonight.” The Scribe quietly writes a note — Events/forge-watch = “Mira promised to guard the forge tonight” — so 50 messages later the AI still knows about the promise.

Just want it to work? Leave the defaults. These settings only change how it takes notes (e.g. which model does it, or asking you to review before saving).
