🧠 How this works — click for a simple explanation

In plain words: a background Memory Agent reads what just happened after each reply and saves the lasting facts — names, promises, injuries, secrets. It keeps a compact memory sheet up to date: the story so far, the current scene, and the facts the next reply needs.

Before each reply, that sheet is handed to your main AI by pure code — no extra AI call, so your replies are never slowed down. Your main AI never uses tools; it just reads the sheet.

Just want it to work? Tick Enable and pick a cheap model below for the Memory Agent. Leave everything else at its default.

What actually gets sent (assembly order)

The Memory Agent runs a background tool-loop session. The SYSTEM message is the fixed built-in rulebook (tool protocol + extraction rules). The USER message carries the live data:

[SYSTEM] built-in Memory Agent rulebook (fixed)

[USER]
## Task (full run or extract-only)
## Observation date / Character Info / User Persona
## Current scene + Story so far (reflection)
## Prior memory sheet
## Memory database overview (drawers + key inventory)
## Taxonomy menu
## SETTLED messages (extraction sources)
## TENTATIVE messages (planning context only)
## Additional instructions from the user (the field above)

The agent then calls tools (list_categories → list_keys → read_facts → write_fact / search, max 6 rounds / 20 calls) and finishes with the #SHEET block that becomes the memory sheet.
